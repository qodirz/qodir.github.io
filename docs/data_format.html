Data Format

We are expecting the incoming files to be in standart utf-8 encoded CSV format. in following sections we have provided information about the required data fields and their format that Bridg typically ingests from a client's data warehouse or a third party system. we'll use this document to:

  • Gain a better understanding of what type of data you should be sending to Bridg.
  • Identify data gaps or potential augmentation opportunities in existing data sources.
  • Develop a schema to transfer data to Bridg.

At a high level, the daily data should include:

  • Check Summary
  • Tender
  • Line item
  • Discount
  • Comp
  • Location
  • Customer
  • Dimension files
    • Tender Type dimension
    • Tender Type detail dimension
    • Order Source dimension
    • Discount dimension
    • Comp dimension
    • Category dimension

Please Confirm which fields you have available and will be sending. if a field won't be available or sent, Bridg needs to agree and confirm that we can proceed without it.
